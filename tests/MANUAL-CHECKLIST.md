# 手动测试清单

## 加载扩展
1. Chrome/Edge 打开 `chrome://extensions`
2. 打开右上角「开发者模式」
3. 「加载已解压的扩展程序」选择项目根目录 `trans-web/`
4. 确认无报错，工具栏出现扩展图标

## 配置
- [ ] 点图标 → 弹窗 →「打开设置」
- [ ] 选 provider（OpenAI 兼容 / Anthropic），填 Base URL、API Key、模型名，保存 → 显示「已保存 ✓」
- [ ] 切换 provider 时，空的 Base URL 自动填对应默认值

## 翻译此页（test-page.html）
打开 `tests/test-page.html`（file:// 或本地服务器）：
- [ ] 点图标 →「翻译此页」：译文在原文「下方」另起一行（不是并排/同一行），先显示「翻译中…」再变译文
- [ ] 译文字体与原文一致（同一种 font-family），字号为正常正文大小（不照搬标题的大号粗体）
- [ ] `<pre><code>` 代码块**未**被翻译
- [ ] `123 456 789` 与 `$ 1,000.00` 纯数字/符号段落**未**被翻译
- [ ] 滚动到底部的远端段落，进入视口后才被翻译（懒加载）
- [ ] 「编辑器风格」嵌套 div+span 段落（模拟飞书/Notion）也能翻译（不是只认 `<p>`）
- [ ] Flex 标题的译文在其「下方」另起一行，而不是排在旁边
- [ ] 再点一次「恢复原文」：所有译文消失，恢复原状

## 飞书/SPA 实测（需登录）
- [ ] 打开飞书文档 wiki 页，开译后正文（div 排版）能出现译文
- [ ] 向下滚动，懒渲染/虚拟滚动新出现的段落也会被翻译（MutationObserver 跟踪）

## 触发方式
- [ ] 快捷键 Alt+A 可切换翻译/恢复
- [ ] 在设置里把测试域名加入「自动翻译域名」，重开该页 → 自动开始翻译

## 错误处理
- [ ] 清空 API Key 后翻译 → 页内提示「请先在扩展设置里填写 API Key」
- [ ] 填错误 Key/URL 翻译 → 右下角红色 toast 显示状态码与错误信息，译文占位被移除

## 单元测试
- [ ] 根目录运行 `npm test`，translator 与 segment 全部通过
